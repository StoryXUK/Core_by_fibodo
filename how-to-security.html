Frequently Asked Questions

Everything you need to know about fibodo Security.

GDPR-compliant for personal data processing in the EU/UK. PCI DSS compliant for card payments. Industry-standard practices aligned with ISO 27001 principles.
In transit: TLS 1.2+ encryption for all client-server communication. At rest: AES-256 encryption on databases, backups, and storage. Encryption keys are rotated regularly and securely managed.
Each customer's workloads run in isolated containers. Containers prevent cross-account access, ensuring data separation. Provides scalable, secure deployments with minimal risk of data leakage.
Role-based access control (RBAC) limits visibility and actions based on role. Admins can assign granular permissions per module (bookings, payments, reports, etc.). Access logs track every login, modification, and sensitive action.
Yes, multi-factor authentication (MFA) is supported for all staff and admins. Supports authenticator apps, SMS codes, and security keys.
Regular patching schedule for OS, frameworks, and dependencies. Critical vulnerabilities are patched immediately. Updates are tested in a staging environment before production rollout.
Continuous monitoring with IDS/IPS systems and anomaly detection. Suspicious activity triggers alerts to the security team. Automated blocking and throttling for suspicious IPs.
1. Detection & triage by security team 2. Containment and mitigation of affected systems 3. Root cause analysis and recovery 4. Customer notification if data is impacted (per GDPR/PCI rules) 5. Post-incident review and preventive measures.
Hosted in ISO-certified cloud data centres. Location options available in EU, UK, and US depending on your account. Data residency rules comply with GDPR, CCPA, and other regional regulations.
Report via the security contact email or portal. Fibodo operates a responsible disclosure program and responds promptly. Critical issues may be eligible for recognition under bug bounty guidelines.
